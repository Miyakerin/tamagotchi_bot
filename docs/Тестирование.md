## Ручное тестирование

Ручное тестирование проводится для проверки функциональности, которую сложно автоматизировать или которая требует человеческой оценки. Этот процесс включает в себя выполнение тестов вручную по заранее подготовленным сценариям.

### Таблица 1: Функциональное тестирование

| **Тест-кейс**             | **Цель**                                   | **Шаги**                                                                                                                                      | **Ожидаемый результат**                                                    |
|---------------------------|--------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------|
| Создание Тамагочи         | Убедиться, что пользователь может создать Тамагочи | 1. Пользователь отправляет команду для создания Тамагочи.<br>2. Бот проверяет лимит тамагочи для пользователя.<br>3. Если лимит превышается, бот выдает сообщение о невозможности добавить тамагочи.<br>4. В ином случае бот подтверждает создание Тамагочи и показывает его статус. | Тамагочи создан, данные сохранены в базе данных, статус отображается корректно. |
| Просмотр состояния Тамагочи | Убедиться, что пользователи могут видеть текущее состояние Тамагочи | 1. Пользователь отправляет команду для просмотра состояния Тамагочи.<br>2. Бот отображает текущие параметры Тамагочи. | Состояние Тамагочи отображается корректно.                                  |
| Обновление состояния Тамагочи | Убедиться, что взаимодействие с Тамагочи обновляет его состояние | 1. Пользователь отправляет команду для кормления/игры с Тамагочи.<br>2. Бот обновляет состояние (уменьшение голода/увеличение счастья).<br>3. Бот отображает обновленный статус. | Голод Тамагочи уменьшен/Счастье увеличено, статус отображается корректно.                    |
| Использование предметов   | Убедиться, что предметы могут быть использованы для улучшения состояния Тамагочи | 1. Пользователь отправляет команду для использования предмета.<br>2. Бот применяет эффект предмета.<br>3. Бот отображает обновленный статус. | Эффект предмета применен, статус Тамагочи обновлен.                        |

## Автотесты
Ознакомиться с используемым актуальным пайплайном можно тут: [Пайплайн](.gitlab-ci.yml)

Стадии пайплайна:

- build: включает в себя этап сборки приложения
- test: включает в себя unit-test-job и lint-test-job. unit-test-job отвечает за тестирование функциональности кода, когда как lint-test-job отвечает за проверку синтаксиса кода.
- deploy: включает в себя процесс развертки продукта в продакшн.
